> Прототип 3D‑механики: изометрическая сцена с производственными зданиями, сбор ресурсов персонажем, независимое перемещение камеры. Проект поддерживает **PC / WebGL / Android** и выполнен на **Unity 6000.0.59f2**.

---

## Ключевые возможности

* Пять зданий‑производств с независимыми циклами генерации ресурсов.
* Point‑and‑Click перемещение: клик по земле — движение в точку; клик по зданию — подходим и собираем.
* Свободная изометрическая камера, независимая от движения персонажа.
* Всплывающий **суммарный попап ресурсов** (агрегация собранного за сессию/период).
* **Настройки звука** с автосохранением между сессиями.
* **Сохранение прогресса**: произведённые и собранные ресурсы (JSON).
* Расширяемая модель ресурсов (набор типов задаётся конфигурацией).
* Машина состояний для управления игровыми режимами/экранами.

## Архитектура

### Общая идея

В основе — **машина состояний игры** (Game State Machine), которая управляет экранами/режимами и разделяет ответственность: загрузка сцены, игровой цикл, пауза, показ результатов.

Проект разбит на **Infrastructure** необходим для общей логики (сохранения, управления сценами) и **GameObjects** логика непосредственно игры (Player, Camera)

Настройки вынесенны в SO объекты в папке  **Assets/_SO_Settings**

### Ключевые компоненты

* `GameStateMachine` — машина состояний игры.
* `ISceneManager` — загрузка сцен.
* `ISaveManager` — сохранение и загрузка данных.
* `ISaveStorage` — запись / чтение сохранения на устройство.
